Patata.io

Feature: Patata.io framework
As a developer, QA, or stakeholder
I want a BDD testing framework
So that testing apps becomes easy as Given, When Then.

Early beta access

Scenario: Accessing to beta
Given I am on the patata.io landing page
When entering an email address to the subscription form
Then I should soon receive an invite to early access the awesome Potato.io framework.

One-click install

Scenario: Installation
Given a terminal
When I write "npm install patata --save"
Then I should have the entire framework with all dependencies installed on my computer.

Reports (HTML, Json, XML, ...)

Scenario: Creating reports
Given a community report-plugin for Patata.io
When I execute my tests
Then I should get a report defined by the plugin.

Integrations (Jira, Slack, Bamboo, TeamCity, ...)

Scenario: Continuous Integrations
Given a community ci-plugin for Patata.io
When I execute my tests
Then my CI tool should obtain and understand the results.

Testing made easy (Apps, Environments, Branches)

Scenario: Testing
Given a terminal
When launching a test with attached flags such as --app1 --prod --rel
Then my test suite will target proper app, environment, and branch.

Documentation

Scenario: Documentation
Given the "http://patata.io" website
When I access to the documentation section
Then I should easily find documentation of the Patata.io usage.
